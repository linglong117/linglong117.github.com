中智商展携手O2O众巨头, 重塑小微会议生态圈

查看原文

经过长期沟通与准备，中智商展集团近日与全球领先的生活服务电商平台-美团-大众点评正式签署了战略合作协议，并携手滴滴、神州、以及其它O2O巨头，全面启动了资源及系统平台对接计划，基于国际知名会计师事务所及律所的合规咨询指导，以全新模式与技术打通小微会议资源覆盖不足、供应商管理缺失、操作结算繁复、数据分散失准及合规监管低效的瓶颈。在国家相关机构的大力扶持下，中智商展集团有计划加速拓展全国服务网络的建设，颠覆传统模式，重塑企业，尤其是医药企业海量小微会议生态圈。

小微会议，也称小型会，包括城市会，销售会等多种类型，是MICE行业对分布各地高频次小规模会议及培训活动的统称，其规模堪称海量。相对市场会议而言，此类会议金额不大，日程简单，但高频次分布于全国数百个城市，资源需求量极大，之前没有任何一家MICE服务公司有能力满足用户需求并完成全国服务覆盖，更谈不上为终端客户提供高效便捷的工具，且往往由于缺乏经济有效的监管手段，合规问题异常突出，给企业自身带来极大风险。

 中智商展集团在国内第一个提出“智慧会展”理念，率先开启MICE行业3.0时代。在2015年成功上线了“智会”多供应商会议协作及监管云平台，经过磨合，已在辉瑞、拜耳、IBM、联想、AIA、思科、西门子、NetApp、CIMA、LG、中欧商学院等多个客户近千场会议中广泛使用，去年年底更是在某企业6000人大会上被全面应用，满意度超过94%。

“海量小微会议解决方案”是从“智会”云平台中衍生出的又一个重要产品线，它基于平台化服务思维，通过PC及移动端智能应用，对接O2O系统，管理整合海量资源与供应商，同时配合中智线下服务网络体系，创造出全新服务模式。不仅解决了小微会议中的餐饮、酒店、交通、人员、礼品服务等资源的区域覆盖、快捷预定、合规监管、数据归集等问题，还大幅降低了运营成本，提升了用户的体验。目前，此平台产品已在几个医药及IT龙头企业进行战略试点，中智商展与美团-大众点评、神州、滴滴及企业客户一起紧密合作，不断对产品进行完善与升级。

中智商展集团董事长兼总裁陈伟表示，实现上述模式绝非易事，至少需要MICE服务企业具备六个前提条件：

1．规范经营的意愿与能力：

作为第三代MICE服务公司，中智商展集团明确提出合规经营是MICE企业未来发展方向，并通过智慧化系统主动严控内部与外部操作规范，通过模式与技术创新堂堂正正赢得收益。合规，早已成为中智商展集团的企业文化；

2．对客户深入的了解：

中智商展积累了超过20年服务企业客户的成功经验，尤其了解各类国际跨国公司的服务需求、标准、流程、文化甚至习惯，往往从战略的角度，深度介入客户的会议管理与执行，为客户提供精准的服务产品与贴心的增值服务；

3．扎实的线下服务基础：

不论线上平台如何完善，MICE服务都无法脱离线下团队的支持与补充。中智商展集团长期的会议执行服务，积累了丰厚的实战经验，打磨出覆盖国内外数百个热点区域的专业服务网络，如此积淀，绝非一日之功可成。

4．强大的技术解决能力：

中智商展是国内唯一一家投重资组建专属IT公司的集团，并敢于持续投入支持创新研发，通过长期的锤炼与结构调整，研发团队已成为既了解MICE行业，又具有全面自主研发能力的精锐之师，综合实力不可小觑。

5．互赢的战略合作伙伴：

美团-大众点评、神州、滴滴及其它O2O企业经过近几年高速拓展，已具备了巨大的资源优势，占据市场主要的份额。此外，一些国际知名IT及咨询公司也都在各自领域保有领先优势。中智商展集团与这些企业强强联手，互补共赢，可以更好的满足客户不断变化的需求，迅速推出适应市场的服务产品。

6．国家机构的扶持:

中智商展服务网络的建立得到了国家相关机构的充分认可，并给予积极的配合与政策扶持，这是中智商展集团完成标准化及全国服务网络覆盖，实现“智慧会展”构想的有力保障。

近期，中智商展集团旗下会展服务板块公司已完成股份制改造，计划率先在证券市场挂牌，这将进一步加速中智商展正规化、规模化、智慧化目标的实现，助力中智商展成为中国最具实力与规模的第三代MICE服务集团。

智慧，让未来提前到来！

hexo deply提示You should configure deployment settings in _config.yml first! Available Types: git...

问题描述：

hexo d -g 提示：

You should configure deployment settings in _config.yml first!

Available deployer plugins:
git

For more help, you can check the online docs: http://hexo.io/

按照文档一步一步来的，没有添加新文章，就修改了deploy 项，npm install hexo-deployer-git –save 也安装了 hexo-deployer-git还是不行，用百度花了不是时间还是不知道问题出在哪，文件夹都删除了好几次，后来用了google 才发现问题的所在，问题链接：
https://github.com/hexojs/hexo/issues/1263 请教解决方法 #1263
yml格式要求严格，:之后只能有一个空格。这个检查了好几遍没发现问题；但是 拷贝复制deploy 的时候 格式出错了：我是直接从 https://hexo.io/docs/deployment.html 官方文档上复制的deploy 格式，黏贴的时候格式如下：

deploy:
type: git
repository: https://github.com/XXX/XXX.github.io.git
branch: master

解决方法：

假如deploy后面的三个参数，正确的格式应该这样：
deploy:
type: git
repository: https://github.com/tongxindao/tongxindao.github.io.git
branch: master

变量名之前两个空格，参数值前一个空格，用:分割

hexo 提交导致github page仓库上另外添加的文件被删除

问题描述：

大家好，我申请了一个域名，然后用hexo生成静态页面并部署到github。

然后我的域名要定向到github上必须在xxx.github.io仓库的根目录添加一个文件CNAME，其中内容是我申请的域名。

但是莫名其妙的是

每次执行hexo g和hexo d之后，这个CNAME的文件都被删除掉了…导致我无法通过域名访问github page..

解决方法：

确保你是在hexo的source目录下添加CNAME，这样每次构建时才会对应在仓库的根目录生成CNAME；然后再 hexo g和 hexo d；
